# Data Dictionary

This data dictionary describes the project as implemented in the repository. It is grounded in the notebook code, `src/` helper modules, dbt models, and the generated dbt artifacts in `target/`.

Artifact snapshot used for row counts: `target/catalog.json`, generated by dbt on `2026-04-26`. Row counts should be treated as a recent artifact snapshot, not as a live BigQuery guarantee.

## Layer Overview

| Layer | BigQuery schema | Owner in code | Purpose |
| --- | --- | --- | --- |
| Raw | `raw_market` | `01_merval_analysis.ipynb` | Yahoo Finance landing tables for stock and factor prices. |
| Processed | `processed_market` | `01_merval_analysis.ipynb`, `02_transform_features_metrics.ipynb` | Standardized base prices, return features, stock metrics, beta metrics, and correlations. |
| Analytics | `analytics_market` | `03_transform_serve_montecarlo.ipynb` | Basket definitions, horizon definitions, optimized weights, portfolio metrics, Monte Carlo outputs, recommendations, and metric dictionary. |
| dbt processed | `dbt_processed_market` | dbt staging/intermediate models | dbt views/tables that standardize, reconcile, and enrich notebook-created tables. |
| Serving | `serving_market` | dbt marts | Final reporting and presentation-facing marts. |

## Common Identifiers and Metadata

| Column | Type | Meaning |
| --- | --- | --- |
| `date` | `DATE` or `DATETIME` | Trading date or observation date. Source notebooks convert dates with pandas; dbt staging casts many date fields to `DATE`. |
| `ticker` | `STRING` | Equity ticker, uppercased in dbt staging. |
| `factor_name` | `STRING` | Factor/proxy name such as `MERVAL`, `USDARS`, `VIX`, `EEM`, or `US10Y`. |
| `basket_name` | `STRING` | Configured basket key from `src/config.py`: `short_term_tactical`, `medium_term_balanced`, or `long_term_structural`. |
| `horizon_name` | `STRING` | Configured horizon key from `src/config.py`: `short_horizon`, `medium_horizon`, or `long_horizon`. |
| `weighting_method` | `STRING` | Portfolio method: `equal_weight`, `max_sharpe`, `min_volatility`, or `risk_parity`. |
| `basket_horizon_id` | `STRING` | Composite key built as `{basket_name}__{horizon_name}__{weighting_method}`. |
| `ingestion_timestamp` | `TIMESTAMP` | Timestamp assigned by the notebook run that created the row. |
| `run_id` | `STRING` | Notebook run identifier, generated as `run_YYYYMMDD_HHMMSS`. |

## dbt Staging and Intermediate Models

### Staging views in `dbt_processed_market`

The staging views standardize source data through casting, trimming, upper/lower casing, filtering null keys, and deduplication where implemented.

| Model | Source | Grain | Main behavior |
| --- | --- | --- | --- |
| `stg_stock_prices` | `raw_market.stock_prices` | `date`, `ticker` | Casts fields, uppercases ticker, keeps latest duplicate by ingestion timestamp/run ID. |
| `stg_factor_prices` | `raw_market.factor_prices` | `date`, `factor_name` | Casts fields, uppercases factor name, keeps latest duplicate by ingestion timestamp/run ID. |
| `stg_asset_returns` | `processed_market.asset_returns` | `date`, `ticker` | Casts return and factor fields, filters null date/ticker/log return. |
| `stg_factor_returns` | `processed_market.factor_returns` | `date`, `factor_name` | Casts factor values/returns, filters null keys and returns. |
| `stg_stock_metrics` | `processed_market.stock_metrics` | `ticker` | Casts stock metric fields and lowercases `stock_type`. |

## Metric Glossary

| Metric | Definition | Implemented source |
| --- | --- | --- |
| `log_return` | Natural log of the price relative between consecutive observations. | Notebook 2. |
| `usd_adjusted_return` | Stock return minus `USDARS` factor return. | Notebook 2. |
| `excess_return` | Stock return minus `risk_free_daily`. | Notebook 2. |
| `average_return` | Mean daily return annualized by multiplying by 252. | `annualize_return()`. |
| `volatility` | Daily return standard deviation annualized by multiplying by `sqrt(252)`. | `annualize_volatility()`. |
| `beta_vs_merval` | Covariance of asset and MERVAL returns divided by MERVAL variance. | `compute_beta()`. |
| `beta_vs_eem` | Covariance of asset and EEM returns divided by EEM variance. | `compute_beta()`. |
| `corr_with_merval` | Pearson correlation with MERVAL returns. | Notebook 2. |
| `corr_with_fx` | Pearson correlation with USDARS returns. | Notebook 2. |
| `sharpe_ratio` / `weighted_sharpe` | Annualized return or excess return divided by annualized volatility. | `sharpe_ratio()`, Notebook 2, Notebook 3. |
| `sortino_ratio` | Annualized excess return divided by annualized downside deviation. | `calculate_sortino()`. |
| `calmar_ratio` | Annualized return divided by absolute maximum drawdown. | `calculate_calmar()`. |
| `max_drawdown` | Worst peak-to-trough loss from cumulative log-return wealth index. | `max_drawdown_from_returns()`. |
| `downside_frequency` | Share of valid daily returns below zero. | `downside_frequency()`. |
| `concentration_risk_hhi` | Sum of squared portfolio weights. | `concentration_hhi()`. |
| `effective_number_of_assets` | `1 / concentration_risk_hhi`. | `effective_number_of_assets()`. |
| `diversification_ratio` | Weighted average asset volatility divided by portfolio volatility. | `diversification_ratio()`. |
| `diversification_effect` | Weighted average asset volatility minus portfolio volatility. | `diversification_effect()`. |
| `contribution_to_return` | Asset weight multiplied by annualized asset return estimate. | `compute_portfolio_contributions()`. |
| `contribution_to_risk_pct` | Component risk contribution divided by total portfolio risk contribution. | `compute_portfolio_contributions()`. |
| `probability_of_loss` | Fraction of simulated final values below the initial value. | `run_monte_carlo()`. |
| `var_95` | Initial value minus the 5th percentile simulated final value. | `run_monte_carlo()`. |
| `cvar_95` | Initial value minus the mean final value in the worst 5% simulation tail. | `run_monte_carlo()`. |
| `selection_score` | Weighted rank score across risk-adjusted return, return, risk control, tail resilience, and diversification components. | `build_method_comparison()`. |
| `tail_resilience_score` | Rank-based score combining probability of loss, CVaR share of initial value, and simulated tail drawdown. | `build_method_comparison()`. |

## Implemented Tests Covering the Dictionary

The dbt project includes tests for:

- Uniqueness of key grains, such as `date` + `ticker`, `date` + `factor_name`, `basket_horizon_id`, and recommendation scope/key.
- Not-null constraints on key identifiers and selected metrics.
- Accepted ranges for weights, correlations, volatility, drawdown, probability of loss, simulation count, and score fields.
- A singular test that basket-horizon weights sum to 1.
- A singular test that correlation self-pairs equal 1.
- A dbt expression test that dbt-calculated effective asset count falls between 1 and `num_assets`.

## Caveats

- This dictionary documents implemented tables and fields. It does not describe unimplemented features.
- `inflation_proxy` and `country_risk_proxy` are present but currently placeholder fields set to `NaN` by notebooks.
- `adj_close` is stored, but implemented return calculations use `close`.
- `analytics_market.monte_carlo_paths` and `analytics_market.metric_data_dictionary` are declared as dbt sources but are not consumed by current dbt models.
- Most core financial calculations are produced by notebooks and Python helpers, not by dbt SQL models.
- Score fields, risk profiles, recommendation reasons, and key-risk text are rule-based and heuristic.
